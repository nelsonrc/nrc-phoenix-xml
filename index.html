    nrc.phoenix.xml

    Advanced XML Utilities for TypeScript
    A powerful and modular library for parsing, validating, and summarizing XML data.

    üöÄ Features

    • XML Parsing & Validation with XSD support
    • Data Summarization with configurable aggregation methods
    • Modular and Scalable Architecture
    • Localization Support (e.g., currency & percentage formatting)
    • Built-in Error Handling and Debugging

    üì¶ Installation

    Install via npm:

    npm install nrc-phoenix-xml

    üõ†Ô∏è Basic Usage Example

    üîπ Extracting XML Summary

    import { XmlSummarizer } from "nrc-phoenix-xml";

    const xmlData = `<transactions>
        <transaction>
            <amount>1000.75</amount>
            <tax>50.25</tax>
        </transaction>
        <transaction>
            <amount>500.50</amount>
            <tax>25.80</tax>
        </transaction>
    </transactions>`;

    const summaryString = "transaction,1,*:Global Summary,amount:Total Amount:1,tax:Total Tax:1";

    const result = XmlSummaryUtils.processXmlSummary(xmlData, summaryString, false, {
        roundDecimals: 2,
        currency: "USD",
        showCurrencySymbol: true,
        sortResults: true,
        asJson: true
    });

    console.log(result);

        "Global Summary": [
            { "name": "Total Amount", "value": "$1,501.25" },

        ]
    }

    Parses a summary configuration string into structured specifications.

    Extracts and aggregates data from XML based on summary specifications.

    Shortcut method that combines parseXmlSummary and getSummaryFromXml.

    üìù Summary String Format

    The summaryString parameter follows a comma-separated structure, defining XML elements and their aggregation behavior.

    üìå Format:

    elementName,summaryOrientation,groupingField,groupingDisplayName,attribute1:Label1:aggregationType,attribute2:Label2:aggregationType

    üîπ Explanation

    • elementName ‚Üí Target parent XML element for summarization.
    • summaryOrientation ‚Üí Reserved for future versions to define how summaries are organized (currently unused).
    • groupingField ‚Üí Defines the field used for grouping. If "*", there is no grouping.
    • groupingDisplayName ‚Üí Specifies the key returned in the result, representing the groupingField. If groupingField == "*", this acts as the general summary name.
    • attribute:Label:aggregationType ‚Üí Specifies attributes, labels, and aggregation methods.

    üìå Supported Aggregation Types

    • sum(1) ‚Üí Adds values together.
    • avg(2) ‚Üí Calculates the average.
    • count(3) ‚Üí Counts occurrences.
    • min(4) ‚Üí Finds the minimum value.
    • max(5) ‚Üí Finds the maximum value.
    • StandardDeviation(6) ‚Üí Calculates the standard derivation.

    üîπ Example Summary String

    transaction,1,portfolio_type:Portfolio type,acquisition_value:Acquisition value:1,market_value:Market value:1
    • transaction ‚Üí XML element being processed.
    • 1 ‚Üí summaryOrientation, reserved for future use.
    • portfolio_type ‚Üí Used for grouping (e.g., "A" and "B").
    • Portfolio type ‚Üí The label returned in the result representing portfolio_type.
    • acquisition_value ‚Üí Summarized field representing acquisition value.
    • Acquisition value ‚Üí Label assigned to acquisition_value in the summary result.
    • market_value ‚Üí Summarized field representing market value.
    • Market value ‚Üí Label assigned to market_value in the summary result.

        "Portfolio type": {
